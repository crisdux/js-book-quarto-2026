1.1 Reto 1.1: Conversión rápida a number

CautionDificultad

Intermedio

¿Qué imprime este código?

const array = [true, 33, 9, "-2"];

const f = (arr) => {
  return arr.map(Number)
}

const result = f(array)
console.log(result)

Opciones:

  • A. [1, 33, 9, -2]
  • B. [boolean, 33, 9, string]
  • C. [null, 33, 9, null]
  • D. [undefined, 33, 9, undefined]

Piensa en cómo Number() convierte diferentes tipos de datos.

1.13 Reto 1.13: Trabajando con undefined

CautionDificultad

Intermedio

¿Cúal de los siguientes ejemplos regresa undefined por consola?

//#1
let a;
console.log(a);

//#2
function f(x) {
  return x;
}
console.log(f());

//#3
const obj= {
  name:"Cris",
}
console.log(obj.age);

//#4
function y(){
  let z = 3;
  if(true){
    z = 4;
  }
}
console.log(y())

Opciones:

  • A. Solo el ejemplo #1
  • B. Ejemplo #2 y Ejemplo #3
  • C. Ejemplo #3 y Ejemplo #4
  • D. Todos los ejemplos
TipPista

undefined es un valor que se asigna por defecto a las variables que no se inicializan entre otros muchos casos.

  • C. true, true, false, false, false
  • D. false, ReferenceError, false, false, true

El objeto Boolean permite convertir valores a tipo boolean basandose en los valores falsy y truthy.

1.15 Reto 1.15: Operadores de corto circuito

CautionDificultad

Intermedio

¿Qué imprime este código?

console.log(([] || {}) && (undefined ?? "") || null);
console.log(0 || false || Symbol("hi") || 2n);
console.log(typeof (undefined || null || 0 || ("0" ?? 0)));
console.log((() => "hi")() || (false && true));

Opciones:

  • A. {}, 2n, number, false
  • B. undefined, Symbol("hi"), string, "hi"
  • C. null, Symbol("hi"), string, "hi"
  • D. null, 0, number, true
TipPista

Los valores falsy y truthy son clave para resolver este reto.
